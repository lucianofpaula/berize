# Inteligência Artificial

## Visão Geral

A IA faz parte da experiência da plataforma, não é um recurso opcional. Cada módulo pode possuir seu próprio especialista.

## Commerce IA

Assistente inteligente especializado em vendas.

### Serviços

| Serviço                      | Responsabilidade                |
| ---------------------------- | ------------------------------- |
| `InsightService`             | Geração de insights automáticos |
| `RecommendationService`      | Cross-sell e up-sell            |
| `SalesAnalysisService`       | Análise de vendas e faturamento |
| `CustomerAnalysisService`    | Perfil e comportamento          |
| `InventoryAnalysisService`   | Análise de estoque              |
| `PromotionSuggestionService` | Sugestão de promoções           |
| `ConversationService`        | Perguntas em linguagem natural  |

### Exemplos de Insights

- Estoque do Shampoo Premium acaba em 6 dias
- Pomada Modeladora vende 38% mais nas sextas
- Clientes que compram Óleo também compram Pomada
- Ticket médio caiu 12% esta semana
- 45 dias sem comprar: recupere estes clientes

## Arquitetura

```typescript
interface LLMProvider {
  generate(prompt: string, context: LLMContext): Promise<LLMResponse>;
  generateStream?(prompt: string, context: LLMContext): AsyncIterable<LLMResponse>;
}
```

Provedores suportados: Groq, OpenAI, Anthropic, modelo local.

## Booking IA

Assistente para gestão de agenda.

Capacidades:

- Previsão de horários vagos
- Sugestão de abertura de horários
- Análise de cancelamentos
- Recomendação de serviços

## Regra

Toda integração de IA deve ser feita através de interfaces. Nunca acoplar a implementação a um único provedor.
